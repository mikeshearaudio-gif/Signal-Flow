1) “Signal path first” layout rule
Every board should visually flow left-to-right or top-to-bottom:
Source → Input → Processing / Routing → Output → Destination
So even if the gear is different, the player learns the same mental model every time.
Example:
* Recording: Mic → Preamp → Insert / Console → Recorder
* Live: Mic → Stagebox → Console → Processor / IEM / PA
* Post: DAW Stem → Interface / Renderer → Print / Monitor
* Broadcast: Mic / Playback → Router → IFB / TX / Record
2) Required path gets priority placement
The jacks used in the required connections should be placed in the clearest visual area of the board.
Do not put required jacks:
* at the bottom edge,
* partly offscreen,
* behind scroll,
* inside dense sections,
* or mixed with unrelated jacks.
Distractors can be around them, but the core path should be readable.
3) “Complete subsection, not full device” rule
This may be the most useful global rule.
Before level 25, don’t show a whole piece of gear unless the full device is needed. Instead, show a complete relevant subsection.
Example:
* Don’t show an entire console rear panel.
* Show a complete Aux Outputs section, with enough auxes to make the task make sense.
* Don’t show a full rack.
* Show a complete In-Ear Monitor / Input section.
That keeps realism without clutter.
4) Minimum believable range rule
When a level uses a numbered channel, show enough surrounding channels to make it believable.
Examples:
* If using Input 5, show at least Inputs 1–8.
* If using Aux 5 L/R, show Aux 1–6 L/R.
* If using Matrix 3, show Matrix 1–4.
* If using Channel 12, show a bank like 9–16, not a lone channel 12.
This avoids “random floating jack” syndrome.
5) One labeling system per board
Each board should use only one label system:
* either labels are built into the gear art,
* or labels are generated by the UI,
* but not both competing with each other.
If the UI adds labels, the gear art should be simpler. If the gear art has strong labels, the UI should only add subtle interaction states.
6) Hint mode should reveal reasoning, not basic identity
Basic jack identity should always be visible:
* 1
* 5
* L
* R
* Input
* Output
Hints should reveal:
* correct targets,
* suggested next step,
* signal-flow explanation,
* why a wrong patch is wrong.
So hints should answer “what should I do?”, not “what is this jack?”
7) Wrong patch feedback should teach the category mistake
Instead of only saying wrong, classify the error.
Examples:
* “You patched an output to an output.”
* “This is the right device, wrong channel.”
* “This is an input, but not the IEM input.”
* “This is a PA path, not a monitor path.”
* “Stereo pair order matters: L must feed L.”
That turns mistakes into learning moments.
8) The to-do list should mirror the board labels exactly
If the board says:
AUX / OUTPUTS / L 5 R
The to-do should say:
Aux 5 L Output → In-Ear Monitor L Input
Avoid alternate wording like FOH Aux 5 Left Output if the board uses Aux. The language should match across:
* level brief,
* to-do list,
* gear labels,
* tooltip,
* reality check,
* hint text.
9) Early boards should have no “concept stacking”
A beginner level should teach one new idea at a time.
Avoid combining:
* stereo pairs,
* auxes,
* processors,
* matrices,
* IEMs,
* stagebox routing,
* and PA routing
all in the same early board.
A cleaner progression might be:
1. mono source to input
2. stereo L/R pair
3. aux output to monitor input
4. processor input/output
5. matrix routing
6. full PA + monitor split
10) Use “locked complexity” tiers
Create fixed board complexity tiers that all environments follow.
Example:
Tier 1 — Intro
* 1–2 gear sections
* 8–12 jacks
* 1–2 required connections
* no scrolling
Tier 2 — Basic
* 2 gear sections
* 12–24 jacks
* 3–4 required connections
* no scrolling if possible
Tier 3 — Intermediate
* 2–3 gear sections
* 24–36 jacks
* 5–6 required connections
* scrolling allowed if necessary
Tier 4 — Advanced
* 3–4 gear sections
* 36–60 jacks
* 7–10 required connections
* distractor gear allowed
Tier 5 — Expert
* realistic larger system
* unused equipment allowed
* troubleshooting / diagnosis emphasis
11) Add a per-level board validation pass
Every level should be checked automatically before release:
* Every required node is visible
* Every required node is clickable
* No required node is offscreen at level start unless scrolling is intentional
* No duplicate labels
* No unused equipment before level 25
* Every stereo pair has both L and R
* Every numbered range is logically complete
* Required cable endpoints are centered on the actual jack
* To-do text matches board labels
This would catch many of the bugs we’ve been seeing.
12) Make the level brief explain the pattern, not just the task
The post-it should teach the concept in one short phrase.
Example: Instead of only: “Patch Aux 5 L/R to IEM input.”
Use: “Stereo monitor sends use matching L/R outputs and inputs.”
Then the to-do list carries the exact patch instructions.
13) Keep “Reality Check” as the real-world lesson
The brief teaches the game goal. The reality check teaches the audio concept.
Example:
* Brief: “Patch Aux 5 L/R to the IEM input.”
* Reality Check: “IEMs need an independent monitor mix so the performer hears a custom feed.”
This separation makes the educational layer cleaner.
14) Build every environment with the same grammar
Every board family should share the same visual grammar:
Category Subcategory Jack ID
Examples:
Live:
* Aux / Outputs / L 5 R
Recording:
* Preamp / Outputs / 1
* Recorder / Inputs / 1
Post:
* DAW Stems / Outputs / L R
* Renderer / Inputs / L R
Broadcast:
* Router / Outputs / IFB 1
* IFB / Inputs / Talent 1
Game Audio:
* Event / Output / Music
* Bus / Input / Music
That consistency will make the whole game feel more deliberate.  Universal Recording Studio Layout Rule
Apply this as a universal build rule for the Recording Studio environment:
Active/required area
* Place the level’s required jacks in the smallest practical number of readable rows.
* Keep labels readable.
* Keep jacks evenly spaced.
* Avoid horizontal scrolling wherever possible.
* Do not over-compress labels or jack spacing to the point of unreadability.
Unused lower patchbay area
* Any remaining space beneath the active jacks should be filled by an unlabeled patchbay continuation.
* These unused rows should visually match the active patchbay hardware.
* Use realistic TT-style jacks by default.
* Use 1/4-inch style jacks only where visually appropriate.
* Do not place any background image behind the patchbay.
* Do not use a screenshot, photo, or ghosted labeled patchbay as the background.
* The lower area should look like physical unused patchbay hardware, not a background layer.

Cable Layering Rule
Preserve the v1.40.53 cable stacking fix:
* Wires must render in front of the patchbay/jack face layer.
* Labels should remain above cables so they stay readable.
* Cable endpoints should feel visually connected to the jack face.
* Do not regress to cables appearing behind the jacks.

Visual Target
The Recording Studio board should read as:
1. Top portion: active level-specific jacks, consolidated and readable
2. Lower portion: unused unlabeled patchbay rows filling the remaining board space
3. No background image: only actual rendered patchbay structure
4. No solution reveal: inactive/unneeded jacks remain decorative only
5. No horizontal scrolling: unless absolutely unavoidable for a larger future level

Do Not Change
Preserve from v1.40.53:
* cable-over-jack stacking behavior
* current realistic jack styling direction
* compact Recording Studio active-jack grouping
* progressive gear tile illumination behavior
* left-side control layout
* top gear map behavior

Acceptance Checklist
v1.40.54 is complete when:
* The bottom of the Recording Studio patchbay is filled with unlabeled unused patchbay rows.
* No background image or ghosted labeled patchbay appears behind the board.
* Active jacks remain readable and evenly spaced.
* Unused jacks are visually consistent with active jacks but unlabeled.
* Wires remain visibly in front of the jack faces.
* Labels remain readable above cables.
* The layout applies to Recording Studio levels universally, not just REC-002.
* The game launches/displays as v1.40.54.
