# Quiz Consolidation Audit v1

## Active Quiz Renderers

- `sfv182RenderQuizLevelDirect` is the direct quiz renderer.
- The later `sfv189` wrapper still owns question generation/order, then delegates through `sfv182RenderQuizLevelDirect`.
- Active quiz card fronts are generated by the latest `sfv182QuizCard` override and render `1` through `5`.
- Active quiz panels now include `data-level-id` from the rendered level object.

## Active Quiz Binders

- `sfv182BindQuiz` is the consolidated binder path.
- `sfv189BindQuiz` delegates to the consolidated binder and only adds its live marker class.
- Question scoring goes through `sfv182AwardQuizQuestion`.
- Completion time bonus goes through `sfv182AwardQuizTimeBonus`.
- Timeout partial completion goes through `sfv182CompleteQuizOnTimeout`.

## Old Scoring Calls

- `sfAwardTimedQuizLedgerScoreOnce` remains for legacy callers, but is neutralized to completion time bonus only.
- The old flat quiz completion award is no longer part of active quiz completion.
- Training-only non-quiz `sfAwardLedgerScoreOnce('training-' + level().id, 500, 25)` calls still exist outside the active quiz path.

## Wrapper Load Paths

- Wrapper `loadLevel` updates dev-shell controls, then delegates non-IR boards to `frame.contentWindow.sfNativeGameShellLoadLevel` when the game is already loaded.
- If delegation is unavailable, wrapper falls back to setting the iframe URL.
- Native game shell `loadGameLevel` sets the in-game selector, dispatches `input` and `change`, then clicks the in-game Load Board button if present.
- Wrapper/hash state is not used by quiz scoring; quiz scoring reads the rendered quiz panel level id first.
